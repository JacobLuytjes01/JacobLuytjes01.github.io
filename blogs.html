StringNum

I had an idea for a number system because integers have a small range and do not have any decimal components. Then, floating point can have rounding errors and cannot represent every possible number. I wanted to use a system that could have near-infinite precision; however, I was unaware of the existence of BigNum systems. So, I started to build.
I wrote the project in C++ to be fast and easily usable in C++ and Python code bases. The numbers are represented internally using strings, as I was more familiar with them than char arrays, and I wanted to be able to add Binary, hexadecimal, or other formats, and not only decimal.
I used test-driven development to help create features and correctly implement the basic math functions. Due to the length of this project, it was useful in debugging a large code base and in determining where and what to fix whenever I encountered a bug.
I have currently implemented addition, subtraction, multiplication, prefix and postfix operators, all relational operators, and some helper functions.
The possible features I could add would be division, and compound assignment operators.
This project was a useful learning experience in building a large-scale system.

Fractional Approximation of PI Calculator

I was looking for something to program as I wanted more projects for my resume and portfolio. Then, after a drought of ideas, I thought about how certain fractions approximate PI, like 22/7 or 335/113, and how it would be interesting to create a program to find the best fractions.
The first major problem I had was that 44/14 would appear when 22/7 was there. I tried a couple of ways to get 44/14 to disappear. First, I type-cast the values into integers and check if the numerator and denominator modulo of the surrounding ranking's numerator and denominator equals zero [check if they are exact matches]. However, this did not work because I wanted the user to be able to use fractional components for both numerator and denominator. Then, I checked if two numbers are extremely close together; they would not show up, but this caused issues when I increased the numbers it checked. It would skip over numbers.
Finally, I made a solution that worked well enough. I took the previous solution, but made the lower bounds get divided by the maximum denominator that would be checked. This solved the problem in a simple way by showing the correct ranking for any depth while removing duplicates.
